Vanguard ETF Analysis Project

Pandas DataFrame visualization of the data collected from the first 9 ETFs in Vanguard's ETF list

Duration: Approximately 1 Week (7/20/23 - 7/27/23)

Objective: To Utilize Web-Scraping And Pandas Data Visualization and Analysis To Determine Which Vanguard ETFs Might Be the Best For Me to Invest In

Type of Project: Individual Project

The purpose of this project was primarily to do some self-studying on web scraping. I've heard a lot about the topic and I wanted to try my hand at it and maybe glean some useful information in the process. So I chose to develop a program capable of collecting and analyzing data about all the ETFs that Vanguard offers. I began by researching the ETF listings and planning the data collection process, identifying which specific pieces of data that I wanted to analyze (number of stocks, ETF performance in the last x years, etc). Then, I looked into the Python library BeautifulSoup, which we had touched upon in my Data Science class for website data extraction. However, it quickly became clear that because Vanguard's ETF pages did not store information in pure HTML, I would not be able to access the data I wanted using solely BeautifulSoup. So I incorporated Selenium WebDriver and used the xpath of each specific data element to extract the information that I wanted. After collecting the data, my program saved the extracted information into a CSV file (download link).

From here, it was classic Pandas data analysis. In a Google Collab notebook, I imported my CSV file into a DataFrame for better visualization. Then, I filtered out entries based on my personal criteria (detailed explanations of my filtering decisions can be found in the Collab Notebook). In the end, I compiled a shortlist of the ETFs that fulfilled my requirements.

If I was going to further fine-tune the project, I would conduct more research on methods to expedite the data collection process. As it stands, the program takes several minutes to run, and this is even after I incorporated the 'headless' option for Firefox Selenium WebDriver and tried to cut down static wait times for page loading by using exception handling instead.
