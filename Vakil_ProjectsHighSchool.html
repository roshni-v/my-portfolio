Virtual Card Game Project

The graphical user interface for my game, where the blue cards have been cleared, the cards you can see are opened, and the faintly checkered cards are closed and yet to be cleared.

Course: AP CSA (AP Computer Science A)

Duration: Approximately 2 Weeks (4/15/19 - 5/02/18)

Objective: To Create a Novel Card Game Given Basic Java Code and a GUI

Type of Project: Individual Project

The goal of this project was to creatively alter the program for a virtual card game called elevens that we were guided through coding in class. In this elevens card game, two virtual cards can only be cleared if they are the only cards selected and their sum is eleven. From this, I chose to create a game called Clear the Board. This game is quite like memory game except it is Single Player, you can flip over as many cards as you want at a time (but you can only remove cards from the board if you close all other cards except for the pair that you want to clear), and there is no winner. The goal, instead, is to clear the board in the fastest amount of time. My development process for this program was three-fold. First, in the ElevensBoard class, I changed the program to recognize pairs of the same point value instead of pairs that add up to eleven. Second, I called only clubs and diamonds onto the board in the Deck class so that there are only two cards with the same point value and not four. I also added two jokers. Finally, I altered the CardGameGUI. The majority of my time working on the project was spent here. I changed the logic for the display and removal of cards and added a timer. My main challenges were understanding the provided starter code, figuring out how to be able to open and close the cards and clear them if a double arised, and adding a timer. Potential future improvements include setting the program up so that you can only open two cards at a time and making the code itself more concise.

Interactive Seating Chart Project

My class' static classroom class.

Code for my circleClass() method.

Course: AP CSA (AP Computer Science A)

Duration: Approximately 1 Month (8/14/18 - 9/04/18)

Objective: To Learn Object Oriented Concepts of Inheritance and Data Encapsulation

Type of Project: Individual Project

The task assigned to my entire AP CSA class was for each student to create a class named after themselves in Greenfoot, an integrated development environment using Java. This class was to be derived from an abstract student class provided to us, and was to use a static classroom instance to do the following: encompass each student instance and visually display their distinct location in the classroom. The location was based on our actual seating arrangements in class. My RoshniVakil class had an image of myself, an audio recording of me saying my name, and an action method that I constructed called circleClass(). When my image in the classroom was clicked, my audio recording would play and my circleClass() method would cause my image to spin. I worked through this project by first understanding what was provided to me. Then I began to piece together my RoshniVakil class by recording my name pronunciation file and downloading a photo of myself. The most challenging part of this project was programming the circleClass() method. It was where we were given complete creative will to have our photos do anything in the span of roughly ten seconds. I chose to have my image spin, turn into a capital R, and then reorient itself upright before returning to the photo of my face. It took several iterations to perfect the entire sequence.

Hangman Project

The ASCII art displayed if the player guesses the phrase within 15 tries.

This function that constructs the dashes, spaces, and punctuation (representing the secret message) first displayed to the player.

Course: CSE (Computer Science/Software Engineering)

Duration: Approximately 1 Week (2/07/17 - 2/14/17)

Objective: To Model a Hangman Game Using Python Coding

Type of Project: Pair Project

For the Hangman Project, my partner and I were given a set of directions which instructed us to model a Hangman Game in Python. To begin the project, we created a basic function that took a secret message and set of letters and checked to see whether or not the letters occurred in the secret message. If so, the letters were returned in a set of dashes and spaces, where the dashes represented the unguessed letters in the secret message. This simple function then became the basis of our final game, which was the most challenging transition to make. My partner and I had to take it step by step and code one feature at a time, for our end product would have been much more difficult to create if we had attempted to program it all at once. Our final game randomly chooses a secret message from a list of messages. Then it displays a series of dashes, spaces, and punctuation to the user (representing the message). The user is then provided fifteen guesses. At any point in the game, they can choose to guess the entire phrase; else they can go letter by letter. Whether or not they succeed, a play again option is provided at the end. Our project earned full points for this assignment because it worked in all situations, had concise and effective code, and accurately represented an actual Hangman game. I also learned how to make ASCII art and incorporated it into our project, going above and beyond the criteria.

Scratch Project

Programming Interface

Course: CSE (Computer Science/Software Engineering)

Duration: Approximately 2.5 Weeks (8/30/16 - 9/16/16)

Objective: To Create a Fully Functioning Game Using Scratch

Type of Project: Pair Project

For my very first ever coding project, my partner and I were given the task of creating a fully functioning game or story using Scratch. From the instructions and rubric provided we decided to create a snail race game based off of the Spongebob episode, The Great Snail Race. After deciding our plot, we started coding. Our work method was to allow one partner to code while the other watched and provided suggestions. After about fifteen minutes, we would switch roles. This method of teamwork was especially effective because it allowed both partners to have an equal time coding and because only by working together were we able to accomplish what we did. For initially, I had found Scratch to be rather confusing, but after some help from my partner and practice at home, I was able to get a better understanding of how coding worked. With that knowledge we made great progress on our program together. In the end, our game featured three players with whom you can play against (each with varying degrees of difficulty). In addition, there is a "You Win" and "You Lose" screen, and a "Play Again" option. These features were accomplished by creating sprites for the three computer-operated opponents and coding them to race under certain conditions. The game might be somewhat glitchy, for it is our first project, but it gave us a valuable introduction to the logic of coding. My partner and I each brought our own skills to the table, which combined helped make this game the best that it could be given our skill level.
